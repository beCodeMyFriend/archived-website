Creo que una de las mejores maneras de entregarle el código a un cliente es dejándoselo en una cuenta de Bitbucket[1] que él mismo se haya creado o hayamos creado para él. De esta forma no sólo tendrá el código, sino tendrá cada uno de los commits que hemos hecho y podrá ir viendo, si quiere, como vamos evolucionando el producto. Esta es otra forma más de ser transparentes con el cliente.

Por otro lado, nos gusta quedarnos con el trabajo que hacemos, siempre que el cliente no tenga inconveniente, así que también usamos nuestro propio repositorio de código.

Normalmente usamos Git para trabajar. Quería poder hacer push en ambos repositorios para tener siempre los 2 actualizados de la misma manera y encontré que editando el fichero .git/config, podría hacerlo Simplemente añadiendo las siguientes líneas:

[remote "all"] url = url_repo_1 url = url_repo_2

De esta manera, si hacemos un git push all se actualizarán automáticamente nuestros 2 respositorios.

Ya se que no es un tip muy pro, pero igual a alguien le viene bien.

[1] Se podría usar GitHub o cualquier otro, pero usamos Bitbucket simplemente porque nos permite tener un repositorio privado sin coste extra para el cliente.
