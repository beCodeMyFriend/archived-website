En este pequeño manual voy a explicar cómo instalar Oracle 10 en un sistema Linux basándome en una instalación como base de Ubuntu 8.04 Server sin ningún paquete opcional. Tan solo la instalación mínima de Ubuntu y, sobre ella, comienzo a explicar los pasos no sólo para instalar Oracle, sino también para poder tener acceso a su administrador remotamente. Ya que, por defecto, Oracle no da esta opción y la administración web sólo es posible desde 127.0.0.1.

Paso 1. Adquirir la clave pública:

wget http://oss.oracle.com/el4/RPM-GPG-KEY-oracle -O- | sudo apt-key add ñ

Paso 2. Modificar fuente de datos del apt nano ñw:

/etc/apt/sources.list.

Y añadimos al final del fichero: deb http://oss.oracle.com/debian unstable main non-free.

Paso 3. Actualizar nuestro repositorio:

apt-get update.

Paso 4. Instalar paquete Oracle:

apt-get install oracle-xe.

Paso 5. Configurarlo:

/etc/init.d/oracle-xe configure. Debiendo tener una salida como esta:

Oracle Database 10g Express Edition Configuration.

-------------------------------------------------

This will configure on-boot properties of Oracle Database 10g Express Edition. The following questions will determine whether the database should be starting upon system boot, the ports it will use, and the passwords that will be used for database accounts. Press to accept the defaults.

Ctrl-C will abort.

Punto en el cual nos piden que especifiquemos el puerto sobre el cual se instalará el administrador web, que por defecto está en el 8080:

Specify the HTTP port that will be used for Oracle Application Express [8080]:

Aquí nos piden el password de SYSTEM y SYS:

Specify a password to be used for database accounts. Note that the same password will be used for SYS and SYSTEM. Oracle recommends the use of different passwords for each database account. This can be done after initial configuration:

Confirm the password:

Punto en el cual nos piden que indiquemos si queremos iniciar el servicio de Oracle en boot, por defecto y (yes - sí):

Do you want Oracle Database 10g Express Edition to be started on boot (y/n) [y]:

Una vez completados todos estos pasos veremos lo siguiente:

Starting Oracle Net Listener...Done

Configuring Database...Done

Starting Oracle Database 10g Express Edition Instance...Done

Installation Completed Successfully.

To access the Database Home Page go to "http://127.0.0.1:8888/apex"

Lo cual quiere decir que ya tenemos nuestro oracle instalado, configurado y listo para usar. Pero sólo administrable desde la URL que se nos indica.

http://127.0.0.1:8888/apex

Paso 6. Habilitar el acceso remoto a la administracion:

cd /usr/lib/oracle/xe/app/oracle/product/10.2.0/server/bin./sqlplus

SYSTEM/XXXXX@XE. Donde las XXXXX representan el password que antes hemos indicado en la configuración de Oracle.

Una vez introducida esa orden nos aparecer al prompt de la consola SQL:

SQL*Plus: Release 10.2.0.1.0 - Production on Tue Nov 30 22:43:51 2010

Copyright (c) 1982, 2005, Oracle. All rights reserved.

Connected to: Oracle Database 10g Express Edition Release 10.2.0.1.0 - Production SQL>
